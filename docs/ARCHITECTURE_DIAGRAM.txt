│                              DATA FLOW                                       │
└─────────────────────────────────────────────────────────────────────────────┘

CONTENT CREATION FLOW:
1. Admin logs into Strapi admin panel
2. Creates blog post with title, content, image, etc.
3. Saves and publishes the post
4. Data stored in SQLite database
5. REST API automatically exposes the content

CONTENT DELIVERY FLOW:
1. User visits frontend (http://localhost:3000)
2. Next.js page loads and calls fetchBlogPosts()
3. Axios makes HTTP GET request to Strapi API
4. Strapi returns JSON data with blog posts
5. React components render the data
6. User sees formatted blog posts in browser

Backend:
  • Strapi v4.25.0 (Headless CMS)
  • Node.js 18+ (Runtime)
  • SQLite 3.x (Database)
  • Better-SQLite3 (Database Driver)

Frontend:
  • Next.js 14.2.0 (React Framework)
  • React 18.2.0 (UI Library)
  • Axios 1.6.0 (HTTP Client)
  • CSS Modules (Styling)

Development:
  • npm (Package Manager)
  • Git (Version Control)
  • VS Code (Recommended IDE)

│                         SECURITY LAYERS                                      │
└─────────────────────────────────────────────────────────────────────────────┘

1. Authentication
   └─ Admin panel protected by username/password

2. Authorization
   └─ Role-based access control (Public vs Admin)

3. API Permissions
   ├─ Public: Read-only (find, findOne)
   └─ Admin: Full CRUD access

4. Environment Variables
   └─ Sensitive data stored in .env files

5. CORS Configuration
   └─ Controlled cross-origin requests
